# 总实验报告内容深度与页数硬性要求

本项目最终提交的正式实验报告必须满足以下硬性要求。Codex 在生成报告时必须把这些要求视为最终验收门槛，而不是写作建议。

## 1. 页数要求

- 最终 PDF 页数 **不得少于 50 页**。
- 页数统计以 `report/build/main.pdf` 的实际页数为准。
- 如果 `python scripts/final_report_gate.py` 检测到 PDF 少于 50 页，最终报告不得提交。
- 不允许通过空白页、大段无关文字、重复截图、重复命令输出来凑页数。

建议页数预算：

| 部分 | 建议页数 | 内容要求 |
|---|---:|---|
| 封面与目录 | 2-3 | 按模板生成封面、自动目录 |
| 实验1：数据表示和等效运算 | 8-10 | 结构体布局、内存观察、编码分析、运行结果、错误修复 |
| 实验2：汇编和 C 的混合编程及优化 | 8-10 | C/ASM 接口、调用约定、Debug/Release 对比、优化实验 |
| 实验3：二进制炸弹 | 8-10 | GDB 调试、反汇编、每阶段推导、控制流分析 |
| 实验4：缓冲区溢出攻击 | 8-10 | 栈帧、返回地址、攻击字符串、GDB 验证、安全理解 |
| 实验5：ELF 文件与程序链接 | 8-10 | ELF 节、符号表、重定位、静态/动态链接对比 |
| 实验总结 | 2-3 | 不少于 400 字，建议 800-1200 字 |
| 附录 | 8-12 | 命令、输出、关键代码、GDB/readelf/objdump 摘录、Git 记录 |
| 合计 | 52-65 | 以有效内容支撑页数 |

## 2. 内容充实要求

每个实验正文至少应包含：

1. **实验概述**：说明实验目的、意义、目标和实验要求，不得只复制任务书。
2. **实验内容**：按阶段列出任务，说明每阶段输入、输出和完成标准。
3. **实验设计**：说明思路、技术路线、关键原理、工具选择和预期结果。
4. **实验过程**：必须结合真实命令、调试动作、截图索引、错误修复过程展开。
5. **实验结果**：必须给出运行结果、关键截图、数据或输出文件，并解释结果含义。
6. **实验小结**：总结本实验对应的理论、方法、困难、解决方式和收获。

## 3. 每个实验的最低材料量

每个实验正式写作前，应至少准备：

| 类型 | 每实验最低要求 |
|---|---:|
| 命令记录 | ≥ 8 条关键命令或操作记录 |
| 输出文件 | ≥ 4 个关键输出文件或结果摘录 |
| 截图 | ≥ 4 张关键截图或可替代的终端输出证据 |
| 图表 | ≥ 2 张结构图/流程图/结果表 |
| 错误与修复 | ≥ 1 条真实问题记录，若实验顺利则记录易错点和验证方式 |
| 代码/汇编/反汇编解释 | ≥ 2 段关键代码或指令序列分析 |
| 结论证据映射 | 所有关键结论必须在 `evidence_manifest.md` 中找到证据 |

如果某实验确实不需要截图，可以用完整命令输出、调试日志、反汇编日志替代，但必须在 `evidence/screenshots_index.md` 或 `evidence/evidence_manifest.md` 中说明原因。

## 4. 禁止水字数规则

以下内容不得作为有效页数来源：

- 大段复制任务书而不分析。
- 大段粘贴完整命令输出而不解释。
- 重复放置同一截图或高度相似截图。
- 用空白页、巨大行距、巨大图片、无意义标题撑页数。
- 把常识性概念反复解释，例如反复说明“GDB 是调试工具”。
- 把 AI 使用说明写入正式实验报告正文。

每一页原则上至少承担以下一种功能：

- 说明实验原理；
- 记录实验过程；
- 展示实验结果；
- 分析代码、汇编、ELF、栈帧或调试现象；
- 对比不同方案或不同结果；
- 汇总可追溯证据。

## 5. 附录要求

最终报告必须包含附录。附录至少包括：

1. 关键命令与证据索引；
2. 关键代码与汇编/反汇编摘录；
3. GDB、readelf、objdump 等调试与分析日志摘录；
4. 图表与截图索引；
5. Git 提交记录摘要。

附录不是垃圾箱。附录中的内容必须能支撑正文结论，并且正文应在适当位置引用附录中的证据编号。

## 6. 最终验收命令

正式提交前必须执行：

```powershell
python scripts/final_report_gate.py
```

该脚本会检查：

- PDF 是否存在；
- PDF 是否不少于 50 页；
- 是否存在附录；
- 是否存在实验总结；
- 是否存在 AI 使用记录且未写入正式报告正文；
- 是否仍包含 `待填写`、`TODO` 等未完成占位符；
- 是否存在关键证据清单文件。

只有脚本通过，报告才进入最终提交状态。

## 不允许用猜测补充内容

50 页以上要求必须通过真实实验过程、有效分析、图表、附录和证据材料满足。不得通过猜测个人信息、编造实验结果、重复截图、堆空话或生成无来源附录来增加页数。
