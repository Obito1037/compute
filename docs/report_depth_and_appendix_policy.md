# 报告深度、页数和附录策略

本文件用于指导 Codex 和人工写作者把实验材料整理成一份内容充实、可追溯、页数充足的总实验报告。

## 总体原则

报告页数目标不是机械凑页，而是通过“实验事实 + 原理解释 + 操作过程 + 结果分析 + 证据附录”自然达到 50 页以上。

## 单实验推荐结构

每个实验正文建议采用以下扩展结构：

```text
x.1 实验概述
  x.1.1 实验背景与目标
  x.1.2 实验环境与工具
  x.1.3 实验任务拆解

x.2 实验内容
  x.2.1 阶段一：任务描述
  x.2.2 阶段二：任务描述
  ...

x.3 实验设计
  x.3.1 总体思路
  x.3.2 关键原理
  x.3.3 数据结构/栈帧/ELF/调用约定/控制流分析
  x.3.4 验证方法

x.4 实验过程
  x.4.1 环境配置
  x.4.2 编译/运行/调试步骤
  x.4.3 错误与修复
  x.4.4 关键截图和命令输出说明

x.5 实验结果
  x.5.1 运行结果
  x.5.2 结果解释
  x.5.3 与预期对比
  x.5.4 证据索引

x.6 实验小结
  x.6.1 本实验掌握的理论
  x.6.2 本实验掌握的工具
  x.6.3 遇到的问题与收获
```

正式报告可在保留模板一级结构的前提下，按需要增加三级标题。

## 每个实验的差异化充实方向

### 实验1：数据表示和等效运算

重点补充：

- 结构体字段、对齐、填充字节、内存布局；
- 压缩/解压方案与字节序；
- 整数位运算、浮点数表示、补码和 IEEE 754 相关分析；
- Visual Studio 内存窗口或输出窗口证据；
- 关键测试用例和边界情况。

### 实验2：汇编和 C 的混合编程及优化

重点补充：

- C 与汇编之间的函数接口；
- 参数传递、返回值、寄存器使用、栈帧变化；
- Debug 与 Release 反汇编差异；
- 优化前后运行时间对比；
- MASM 工程配置过程和常见错误。

### 实验3：二进制程序破解

重点补充：

- GDB 基本操作和断点设置；
- 每个 phase 的反汇编片段、条件跳转和输入推导；
- 寄存器、栈、内存观察；
- 控制流图和调试流程图；
- 最终输入的来源说明，避免只给答案。

### 实验4：缓冲区溢出攻击

重点补充：

- getbuf 等函数的栈帧结构；
- 缓冲区起始地址、保存 rbp、返回地址的位置关系；
- smoke/fizz 等目标函数地址；
- 攻击字符串构造过程；
- GDB 验证过程和安全启示。

### 实验5：ELF 文件与程序链接

重点补充：

- GCC 编译流程：预处理、编译、汇编、链接；
- ELF 节头表、符号表、重定位表；
- readelf / objdump / hexdump 输出解释；
- 静态链接和动态链接差异；
- main_sub 与 sub_main 等对比实验。

## 附录组织建议

附录应按证据类型组织，而不是把所有内容无序堆在一起：

```text
附录A：关键命令与证据索引
附录B：关键代码与汇编摘录
附录C：GDB 调试记录摘录
附录D：ELF/readelf/objdump 输出摘录
附录E：图表与截图索引
附录F：Git 提交记录摘要
```

正文只摘录必要内容，长输出放附录。正文必须解释附录中的内容如何支撑结论。
