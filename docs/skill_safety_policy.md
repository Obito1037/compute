# Skill 安全策略

## 1. 总原则

本项目允许参考公开 GitHub skill 的组织方式和工作流思想，但不默认信任任何第三方脚本。所有外部 skill 必须经过人工审查后才能启用。

## 2. 禁止行为

Codex 不得在未获得明确确认时执行：

- 删除项目目录或原始材料；
- 修改 Git 远程地址；
- 强制推送或重写 Git 历史；
- 上传实验代码、报告、截图到未知服务；
- 读取 SSH 私钥、浏览器数据、系统凭证、`.env`；
- 运行 `curl | sh`、`wget | bash` 形式的命令；
- 自动安装不明依赖；
- 覆盖正式报告模板。

## 4. 本项目允许的安全操作

- 读取实验任务书；
- 读取源码；
- 编译实验程序；
- 保存命令输出；
- 生成 Mermaid / PlantUML / Graphviz 图表源文件；
- 编译 LaTeX；
- 用 Pandoc 生成 Word；
- 生成 Git 状态说明；
- 在用户确认后执行 Git commit / push。
