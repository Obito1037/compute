# 结果分析记录

实验二工作副本已在 VS2026 x86 环境下完成 Debug 和 Release 两种配置的编译与运行。两个构建均为 0 警告、0 错误。

运行输出确认：

- `struct student size=38`
- `scores` 偏移为 `20`
- `average` 偏移为 `36`
- 第 0 个学生学号为 `U202414558`
- 第 0 个学生姓名以 GBK/CP936 字节 `D5 C5 BC D2 C1 BA 00` 存入 `char sname[8]`
- ASM 平均分基线与 ASM 平均分优化版本均通过 C 版本校验
- 冒泡排序与快速排序最终结果均为降序

## 分析要点

Debug C 版本包含调试辅助和较多栈上临时变量访问，因此平均分函数计时明显高于 Release C 版本。

Release C 版本中，编译器识别了除数为常量 8 的平均值计算，反汇编中使用 `sar eax,3`，没有生成通用整数除法指令。因此不能简单认为手写 ASM 一定比 Release C 快。

ASM 基线版本使用 `idiv` 进行除法，ASM 优化版本使用右移完成除以 8，因此两个构建下 ASM 优化版本都比 ASM 基线版本快。

排序优化的主要收益来自算法层面。冒泡排序保留双重循环，快速排序使用分区递归；本次 `n=256`、`repeat=200` 的计时中，快速排序在 Debug 与 Release 下均明显更快。
