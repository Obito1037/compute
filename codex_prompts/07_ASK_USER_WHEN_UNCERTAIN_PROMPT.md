# 不确定时必须向用户提问 Prompt

当你遇到不确定、缺失、冲突或无法验证的信息时，不要继续猜测，也不要把猜测写进报告或代码注释。

必须使用以下格式停下来提问：

```text
【需要你确认】
当前实验/模块：
当前步骤：
不确定的问题：
我已经检查过的文件/命令：
可能选项：
A.
B.
C.
我的建议：
需要你回复：
```

特别注意：

1. 姓名、学号、专业班级、指导教师、报告日期、GitHub 仓库地址必须来自 `config/report_metadata.yaml` 或用户明确回复。
2. 如果 `config/report_metadata.yaml` 中仍为“待填写”，必须询问用户。
3. 实验结果、GDB 地址、攻击字符串、ELF 解释、截图内容必须来自证据文件。
4. 不允许为了完成任务而自行补全关键缺失信息。
5. 提问后，把问题记录到当前实验的 `evidence/human_questions.md`。
