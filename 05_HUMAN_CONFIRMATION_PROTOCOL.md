# 人工确认协议：不确定时必须停下来问

本项目允许 Codex 协助完成实验记录、证据整理、图表生成和 LaTeX 报告组织，但不允许 Codex 猜测任何关键信息。

## 1. 必须询问的内容

Codex 遇到以下情况必须停止执行并向用户提问：

### 1.1 封面与提交信息

- 姓名
- 学号
- 专业班级
- 指导教师
- 报告日期
- GitHub 仓库地址
- 是否使用真实个人信息

这些信息只能来自 `config/report_metadata.yaml` 或用户明确确认。

### 1.2 实验执行信息

- 不确定该实验应在 Windows、Visual Studio、MSVC/MASM、WSL、gcc、gdb 中哪一个环境执行；
- 任务书与源码目录对应关系不明确；
- 编译或运行结果与预期不一致；
- 需要解释未保存的输出、未登记的截图、未验证的 GDB 地址或栈偏移；
- binary bomb / bufbomb 的输入、地址、cookie、攻击字符串没有证据支撑；
- ELF 节、符号表、重定位表解释无法从 `readelf` / `objdump` 输出中验证。

### 1.3 文件与 Git 操作

- 删除、覆盖、移动原始材料；
- 首次 push 到 GitHub；
- force push；
- 修改 `.gitignore` 导致结果文件可能被遗漏；
- 提交包含个人敏感信息或大体积二进制文件。

### 1.4 报告质量

- 最终 PDF 不足 50 页，但没有足够有效材料补充；
- 报告正文中的结论缺少证据；
- 图表、截图、命令输出缺失；
- 附录材料来源不清楚。

## 2. 提问格式

必须使用以下格式：

```text
【需要你确认】
当前实验/模块：
当前步骤：
不确定的问题：
我已经检查过的文件/命令：
可能选项：
A.
B.
C.
我的建议：
需要你回复：
```

## 3. 记录要求

每个实验目录必须维护：

```text
evidence/human_questions.md
evidence/assumptions.md
```

- `human_questions.md` 记录所有需要用户确认的问题；
- `assumptions.md` 只能记录低风险、可验证的假设；
- 个人信息、实验结果、GDB 地址、攻击字符串、截图内容不得写入 assumptions 作为假设。

## 4. 继续执行条件

用户确认后，Codex 才能继续：

1. 将确认内容写入对应记录文件；
2. 更新相关配置或证据文件；
3. 继续实验或报告生成。

如果用户没有确认，Codex 必须保持占位符或停止该步骤。
